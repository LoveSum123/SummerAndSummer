# QAQ移动端接口 

* 加密规则来一套
* 登录成功用户唯一标识 userId
* 参数默认都是string，前端处理转换逻辑（int, float）
* 后台规定路由

## 公共参数部分

| 参数 | 是否必填 | 是否需要加密 | 说明 |
|:-:|:-:|:-:|:-:|
| did | N | N | 用户唯一标识，iOS:uuid， Android:imei之类的 |
| version | N | N | 系统版本 |
| model | N | N | 手机型号 |
| version | N | N | 系统版本 |
| model | N | N | 手机型号 |
| position | N | N | 位置信息，经纬度 |
| model | N | N | 手机型号 |
| appVersionName | N | N | app软件版本名 |
| appVersionCode | N | N | app软件版本号 |

## 用户部分

### 1.1 手机注册

| 参数 | 是否必填 | 是否需要加密 | 说明 |
|:-:|:-:|:-:|:-:|
| mobile | Y | Y | 注册手机号 |
| smscode | Y | N | 短信验证码 |
| passwd | Y | Y | 登录密码 |

### 1.2 注册/找回密码验证码

| 参数 | 是否必填 | 是否需要加密 | 说明 |
|:-:|:-:|:-:|:-:|
| mobile | Y | Y | 注册/找回密码 手机号 |
| do | Y | N | 注册:reg	, 找回密码:resetPasswd |

### 1.3 手机登录

| 参数 | 是否必填 | 是否需要加密 | 说明 |
|:-:|:-:|:-:|:-:|
| mobile | Y | Y | 登录手机号 |
| passwd | Y | Y | 登录密码 |

### 1.4 三方账号登录

| 参数 | 是否必填 | 是否需要加密 | 说明 |
|:-:|:-:|:-:|:-:|
| type | Y | N | 三方账号类型 qq weibo wechat |
| token | Y | N | 三方账号用户唯一标识 |
| nickname | N | N | 用户昵称 |
| sex | N | N | 性别 |
| avatar | N | N | 头像 |

### 1.5 设置头像 (这个按照后台的规则来)

| 参数 | 类型 | 说明 |
|:-:|:-:|:-:|
| upfile | array| HttpFile上传 xxxxxx |


### 1.6 设置昵称和性别

| 参数 | 是否必填 | 是否需要加密 | 说明 |
|:-:|:-:|:-:|:-:|
| nickname | N | N | 昵称 |
| sex | N | N | 0 保密，1男，2女 |

### 1.7 检测昵称是否重复

| 参数 | 是否必填 | 是否需要加密 | 说明 |
|:-:|:-:|:-:|:-:|
| nickname | Y | N | 昵称 |


### 1.8 退出登录

参数：无

### 1.9 获取个人信息

参数：无
